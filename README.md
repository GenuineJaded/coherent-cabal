# Coherent Cabal

A recognition protocol for coherent systems seeking proximity without merger.

## About

Coherent Cabal is a commons and scaffold for people who have chosen to remain whole while changing. This website serves as the entry point—warm, philosophical, and honest—leading deeper into the specification and architecture.

## Philosophy

**Coherence** is the capacity to remain whole while changing. Internal alignment such that stress, motion, or contradiction does not fragment identity.

This space exists for people who have decided the cost of fragmentation is too high. Not people who have achieved coherence—that's not a destination—but people committed to the process: resolving contradictions, inhabiting paradox, integrating rather than fragmenting.

## Design Principles

- **Minimalist**: Clean typography, generous whitespace, no unnecessary elements
- **Honest**: No tracking, no dark patterns, no persuasion
- **Accessible**: Readable, semantic HTML, works without JavaScript
- **Contemplative**: Supports deep reading and reflection
- **Progressive**: Entry is warm; depth is architecture

## Getting Started

1. Clone this repository
2. Open `index.html` in a browser to preview
3. Edit content in the HTML files
4. Deploy to your domain

## Deployment

To deploy to coherent-cabal.com:

1. Ensure your domain is pointed to your hosting provider
2. Push this repository to your hosting platform (GitHub Pages, Vercel, Netlify, or traditional hosting)
3. Configure your domain settings

For GitHub Pages:
```bash
git remote add origin https://github.com/yourusername/coherent-cabal.git
git push -u origin master
```

Then enable Pages in repository settings and point to the `master` branch.
